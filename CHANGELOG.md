# 更新日志 (Changelog)

所有重要的版本更新都会记录在此文件中。

## [v0.3.0] - 2026-04-24

### 🚀 新功能

- **画布尺寸升级**
  - 画布从 960x540 升级到 1400x800
  - 适配电脑大屏幕，提供更好的游戏体验
  - 按钮和UI元素同步放大

- **镜面操作优化**
  - 旋转死区从 3° 降低到 0.5°，大幅提升瞄准精度
  - 触摸命中区域从 0.5 倍镜长扩大到 0.8 倍镜长
  - 添加鼠标指针样式（crosshair）

### ✨ 改进

- **UI/UX 改进**
  - 画布最大宽度提升至 1400px
  - 按钮尺寸增大（padding: 12px 20px）
  - 按钮字体增大（font-size: 16px）
  - 添加按钮悬停和点击动画效果
  - 画布边框加粗并添加圆角

- **关卡重新设计**
  - **Level 1-4**：单镜面入门，逐步增加镜子到目标的距离
  - **Level 5-10**：双镜面入门，开始引入障碍物
  - **Level 11-13**：双镜面+多目标，增加障碍物数量
  - **Level 14-16**：双镜面+复杂障碍物布局
  - **Level 17-20**：三镜面+多目标，最高难度

- **难度梯度设计**
  - 目标半径递减：Level 1-4 (20px) → Level 5-13 (18-16px) → Level 14-20 (14-10px)
  - 镜子长度递减：Level 1-4 (130px) → Level 17-20 (90-65px)
  - 障碍物数量递增：Level 1-4 (0个) → Level 5-7 (1个) → Level 14-20 (3-4个)

### 🎮 游戏机制

- **四周边框反射**
  - 光线碰到画布四边会自动反射
  - 增加玩法策略性
  - 可以利用墙壁反弹击中目标

- **通关游戏冻结**
  - 击中所有目标后，游戏自动冻结
  - 无法继续操作镜面
  - 需点击"下一关"或"上一关"继续

- **上一关按钮**
  - 新增返回上一关功能
  - 方便玩家重新挑战

### 📊 统计数据

- 20 个关卡全部重新设计
- 画布尺寸增加 88%（从 518,400 到 1,120,000 像素）
- 旋转精度提升 83%（死区从 3° 到 0.5°）
- UI 元素放大 60%

### 🧪 测试

- 所有关卡结构测试通过
- 6 个基础测试用例全部通过

## [v0.2.0] - 2026-04-24

### 🚀 新功能

- **完整的单元测试框架**
  - 集成 Vitest 测试框架
  - 52个单元测试，覆盖核心算法、游戏状态和关卡验证
  - 数学工具测试（23个）：向量运算、反射计算、距离计算等
  - 碰撞检测测试（9个）：射线与镜面/障碍物相交检测
  - 光线追踪测试（8个）：多段反射和目标命中
  - 游戏状态管理测试（8个）：关卡导航、提示系统等
  - 关卡验证测试（4个）：结构完整性和难度递进

- **本地存档系统**
  - 使用 localStorage 自动保存游戏进度
  - 保存内容：当前关卡、已完成关卡列表、音效设置、首次访问标记
  - 应用启动时自动恢复上次进度

- **移动端操作优化**
  - 添加旋转死区（3°）防止触摸抖动
  - 增大触摸命中区域，提升操作体验
  - 活动镜子视觉反馈（白色高亮+发光效果）

### 🐛 问题修复

- **反射算法修复**
  - 修正了法向量计算逻辑，确保光线正确反射
  - 添加同镜重复命中防护，避免卡死在镜面上
  - 优化反射后的偏移量计算，防止数值精度问题

- **数值稳定性**
  - 统一使用 EPS 值处理浮点精度
  - 优化边界检测，减少穿模和抖动

- **碰撞检测修复**
  - **严重bug修复**：光线与平行线段（镜子）相交检测失败
  - 修复了当光线水平向右发射，镜子水平放置时无法检测碰撞的问题
  - 添加平行情况特殊处理：正确计算光线起点到镜子起点的距离
  - 所有20个关卡物理可解性验证通过

- **关卡同质化问题修复**
  - 重新设计所有20个关卡的几何布局
  - 镜子位置分散在不同高度，增加瞄准难度
  - 目标位置多样化，需要策略性旋转镜子
  - 难度递增：单镜子(1-5)→双镜子(6-10)→带障碍物(11-15)→多目标(16-20)

### ✨ 改进

- **关卡数据完善**
  - 所有镜子、目标、障碍物添加唯一ID
  - 增强关卡结构的可追溯性

- **关卡可解性验证**
  - 创建物理可解性验证脚本
  - 使用游戏实际光线追踪逻辑测试
  - 确保所有镜子能被光线击中

- **游戏状态管理**
  - 跟踪活动镜子（activeMirrorId）
  - 优化状态更新逻辑，减少不必要的重渲染
  - 添加音效开关状态管理

- **代码质量**
  - 完善 TypeScript 类型定义
  - 添加错误处理和容错机制
  - 优化代码结构和可读性

### 📦 技术依赖

- 新增：`vitest@^1.6.0` (开发依赖)
